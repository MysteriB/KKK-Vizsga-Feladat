1. Mappa létrehozása a projekt számára ("projekt_01")

2. Belépés a lérehozott mappába

3. A "git init" parancs kiadása

VAGY
	A "git init projekt_01" parancs kiadása, ami létrehozza a mappát

3.5. Felhasználónév megadása "git config --global user.name fhnév"

4. Állományok létrehozása

5. Állományok hozzáadása a projekthez a "git add fájlnév", vagy "git add -A", vagy a "git add ." paranccsal

6. A commit lérehozása:
	git commit
majd megjegyzés fűzése a commithoz a szerkesztőprogramban, azután mentés, kilépés (ESC, :w, :q)

	VAGY:
		git commit -m "Megjegyzés szövege"

	VAGY együtt a hozzáadás és commitolás:
		git commit -am "Megjegyzés szövege"

7. Változtatások lekérdezése:
	git log [--oneline]

7.1 Visszatérés egy fájl előző verziójához:
	git checkout hash_első_karakterei fájlnév
	
7.2 Hozzáadott fájl törlése a staging area-ról:
	git reset HEAD fájlnév

8. Visszatérés a projekt egy előző verziójához, állapotához
	(A HEAD mutató 		átállítása a "master" ágról):
	git checkout hash_első_karakterei|ág_neve

8.1 Új ág létrehozása (ha szükséges, előtte visszatérni egy korábbi állapothoz):
	git checkout -b új_ág_neve
	
8.2 Ág visszairányítása (összefésülés) a "main" ágba (ha a "main" az aktív):
	git merge visszatérő_ág_neve
VAGY (nem teljesen ugyanaz):
	git rebase ág_neve

 Távoli repository megadása: "git remote add origin https://..."
 Branch átnevezése: "git branch -M main"
 Feltöltés: "git push -u origin main"

Ha előtte szinkronizálni kell (git pull):

- git branch --set-upstream-to=origin/<branch> main
- git pull --rebase

A git nem veszi figyelembe a ".gitignore" fájlban található fájlokat, könyvtárakat.
